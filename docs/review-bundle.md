# Diff-Aware Review Bundles

CtxSift can prepare context for any AI reviewer:

```bash
ctxsift --repo . --diff main...HEAD --mode review --format markdown --out review-context.md
```

The review bundle includes:

- diff spec and Git stat
- changed files
- related tests
- related docs
- optional workspace package graph and package-level reasons
- risk hints for auth/security/dependency/secret-sensitive paths
- selected source context
- AI reviewer prompt
- token budget summary

CtxSift does not decide whether a PR is correct. It prepares the repository context a reviewer needs.

## GitHub Actions

The repository includes `action.yml` and `.github/workflows/ctxsift-review.yml`
as artifact-first PR review-context workflows. By default they use only:

```yaml
permissions:
  contents: read
```

The action builds the CLI, generates both `review-context.md` and
`review-context.json`, and uploads them as the `ctxsift-review-context` artifact.

See [GitHub Action](github-action.md) for install examples, inputs, outputs,
private repository guidance, and comment permissions.

## Optional Sticky PR Comment

Sticky PR comments are disabled by default so the workflow keeps `contents: read`
only.

To enable summary comments:

1. Add `pull-requests: write` to the workflow permissions.
2. Set action input `comment: "true"` in your workflow call.
3. Set repository variable `CTXSIFT_REVIEW_COMMENT` to `true`.

The comment is generated by `dist/pr-comment-cli.js`. It posts only a summary:
artifact name, diff, changed-file count, selected files, token count, redaction
count, and risk hints. It does not embed full source chunks in the PR comment.

This is a decoupled behavior:
- `artifact` upload and comment are independent toggles.
- `upload-artifact: "false"` only disables artifact upload.
- `comment: "true"` only enables sticky comment output.

When comment generation is enabled, artifact generation remains unchanged and is
still uploaded by default, so artifact-first workflows keep complete context
regardless of comment permissions.

## Permissions Notes

- 默认模式 artifact-only 不写 PR comment，仅上传 artifact。
- 需要 comment 时设置 `comment: "true"`，并授予 `pull-requests: write`。
- comment 与 artifact 相互独立，`upload-artifact: "false"` 不会影响 comment。
- 建议继续使用 `pull_request` 触发；避免在不可信 fork 场景改为
  `pull_request_target`。

## 发布闭环说明

每轮版本构建完成后执行：

1. `npm run release:publish:print-command`
2. `npm run release:publish -- --skip-tag-check`

第一步用于生成可复现的 release 命令；第二步用于真实发版（需 `gh` 或有效 token）。
